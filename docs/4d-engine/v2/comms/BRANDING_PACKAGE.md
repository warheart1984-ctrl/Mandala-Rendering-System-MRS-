> Drive-G-1: this package describes how we *intend* to speak about the work. It does **not** prove GPU completeness, Unreal RHI depth, or measured frame times.

## 1. Identity

| Element | Declared positioning |
| --- | --- |
| Product name | **FourDRenderer** (v2.0 = architecture / RFC suite generation) |
| Category line | Higher-dimensional rendering architecture (**declared**) |
| Relationship | Extends MRS / RT4D / Engine v1; hosts consume via FourDAdapter (**skeleton**) |
| Not | “World’s first,” “production-ready Unreal 4D,” or a live commercial brand kit |

## 2. Pillars (three)

1. **Authority in \(\mathbb{R}^{4}\)** — Contracts for acceleration, SO(4)-aware shading, and transport stay upstream of hosts.  
2. **Deliberate observation** — Observation Modes decide *how* W becomes visible; projection is policy, not an accident.  
3. **Honest host landing** — Scene3D + lineage for adapters; deep RHI / Nanite / Lumen remain **roadmap**.

## 3. Voice

| Do | Don’t |
| --- | --- |
| Precise, geometric, calm | Hype absolutes (“world’s first,” “seamless,” “proven 4–6 ms”) |
| Prefer declared / partial / roadmap tags | Pretend RFCs = shipped kernels |
| Speak to architects and technical directors | Promise self-serve studio onboarding today |
| Separate creative aspiration from evidence | Invent press emails or fake benchmarks |

**Tone keywords (declared):** exact · dimensional · observational · disciplined · host-aware

## 4. Taglines (evidence-safe)

Use these (or weaker) in public materials:

- “Declared architecture for higher-dimensional rendering.”  
- “Observe in four dimensions. Project with lineage.”  
- “Contracts first. Host blend when evidence allows.”  
- “FourDRenderer — upstream of Scene3D consumers.”

Punchier brainstorming variants live under [`aspirational/BRANDING_TAGLINES.md`](./aspirational/BRANDING_TAGLINES.md) and **must** be rewritten through the redline before external use.
